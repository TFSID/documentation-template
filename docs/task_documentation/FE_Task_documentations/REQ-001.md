# REQ-001: Form Permintaan Paket Custom

## Deskripsi Tugas
Membuat form permintaan paket custom dengan detail lengkap yang memungkinkan calon jemaah untuk mengajukan permintaan paket umrah/haji yang disesuaikan dengan kebutuhan, preferensi, dan budget mereka.

## Tujuan
- Menyediakan interface yang user-friendly untuk pengajuan permintaan paket custom
- Mengumpulkan semua informasi yang diperlukan untuk membuat penawaran yang akurat
- Memastikan proses pengajuan yang smooth dan intuitif

## Deliverables
- Form permintaan paket yang responsif
- Sistem validasi input yang komprehensif
- Fitur upload file untuk dokumen pendukung
- Step wizard untuk pengalaman pengguna yang lebih baik

## Step-by-Step Implementasi

### 1. Desain Form dan User Interface
```
Frontend Components:
├── Multi-step Wizard Form
├── Personal Information Section
├── Travel Preferences Section  
├── Budget & Package Options
├── Document Upload Section
└── Review & Submit Section
```

### 2. Struktur Data yang Dikumpulkan
```
Personal Information:
├── Nama lengkap (wajib)
├── NIK/KTP (wajib)
├── Tanggal lahir (wajib)
├── Jenis kelamin (wajib)
├── Alamat lengkap (wajib)
├── Nomor telepon (wajib)
├── Email (wajib)
├── Status perkawinan
└── Pekerjaan

Travel Preferences:
├── Jumlah Jamaah (wajib)
├── Tanggal keberangkatan (wajib)
├── Durasi perjalanan (wajib)
├── Kota asal keberangkatan (wajib)
├── Kategori paket (hemat/standard/premium/VIP)
├── Tipe akomodasi (hotel bintang 3/4/5)
├── Preferensi makanan (halal, vegetarian, dll)
├── Kebutuhan khusus (medis, accessibility)
└── Aktivitas tambahan yang diinginkan

Budget Information:
├── Budget maksimal (range slider)
├── Preferensi pembayaran (cicilan/lunas)
├── Tanggal pembayaran preferensi
└── sumber informasi tentang program

Additional Requirements:
├── Preferensi tour guide
├── Kebutuhan dokumentasi khusus
├── Preferensi transportasi dalam negeri
└── Permintaan khusus lainnya
```

### 3. Implementasi Form Validation
```javascript
Validation Rules:
├── Required field validation
├── Email format validation
├── Phone number format validation
├── Date validation (tidak di masa lalu)
├── Numeric validation untuk budget
├── File type validation untuk upload
└── Custom business rule validation
```

### 4. Fitur Upload File
```
Document Upload:
├── Fotocopy KTP (format: PDF, JPG, PNG)
├── Pasphoto terbaru (format: JPG, PNG)
├── Kartu Keluarga (format: PDF, JPG, PNG)
├── Surat keterangan sehat (format: PDF)
├── Dokumen pendukung lainnya
└── Maximum file size: 5MB per file
```

### 5. Step Wizard Implementation
```
Step Flow:
1. Personal Information (10 fields)
2. Travel Preferences (8 fields)
3. Budget & Package Options (4 fields)
4. Document Upload (multiple files)
5. Review & Confirmation
6. Submit Request

Progress Indicator:
├── Visual progress bar
├── Step navigation (next/previous)
├── Save as draft functionality
└── Auto-save every 30 seconds
```

## Kriteria Keberhasilan

### Functional Requirements
- [ ] Form dapat diakses melalui semua device (desktop, tablet, mobile)
- [ ] Semua field required ter-validasi dengan benar
- [ ] File upload berfungsi dengan preview dan progress indicator
- [ ] Data tersimpan sebagai draft saat user keluar dari halaman
- [ ] Email confirmation terkirim setelah submission berhasil
- [ ] Request ID otomatis generado untuk tracking

### Technical Requirements
- [ ] Page load time < 3 detik
- [ ] Form validation response time < 500ms
- [ ] File upload progress indicator realtime
- [ ] Responsive design untuk screen 320px - 1920px
- [ ] Cross-browser compatibility (Chrome, Firefox, Safari, Edge)
- [ ] Accessibility compliant (WCAG 2.1 AA)

### User Experience Requirements
- [ ] Clear error messages dalam Bahasa Indonesia
- [ ] Helpful tooltips untuk field yang confusing
- [ ] Loading states yang jelas
- [ ] Success confirmation yang informatif
- [ ] Option untuk save & continue later
- [ ] Clear navigation antara step

### Security Requirements
- [ ] Input sanitization untuk mencegah XSS
- [ ] File type validation dan virus scanning
- [ ] Rate limiting untuk prevent spam submission
- [ ] HTTPS encryption untuk data transmission
- [ ] CSRF protection
- [ ] Data validation di client dan server side

## Teknologi yang Digunakan
- **Frontend**: React.js dengan TypeScript
- **UI Framework**: Tailwind CSS + Headless UI
- **Form Management**: React Hook Form + Zod validation
- **File Upload**: React Dropzone
- **State Management**: Zustand
- **API Client**: Axios
