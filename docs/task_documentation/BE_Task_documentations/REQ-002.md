# REQ-002: Database Schema Paket Request

## Deskripsi Tugas
Merancang dan mengimplementasikan database schema yang optimal untuk menyimpan semua data paket request, termasuk informasi personal jemaah, preferensi travel, budget, dan dokumen pendukung.

## Tujuan
- Menyediakan struktur data yang efisien dan scalable
- Memastikan data integrity dan consistency
- Mendukung complex queries untuk reporting dan analytics
- Optimized untuk performance dengan proper indexing

## Deliverables
- Database design document dengan ERD
- SQL scripts untuk table creation
- Indexes untuk performance optimization
- Relationships dan constraints definition

### 2. Database Schema Design
```sql
-- Main package requests table
CREATE TABLE package_requests (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_number VARCHAR(50) UNIQUE NOT NULL,
    user_id UUID REFERENCES users(id),
    
    -- Request metadata
    status VARCHAR(20) DEFAULT 'draft', -- draft, submitted, under_review, quoted, approved, rejected
    priority VARCHAR(10) DEFAULT 'medium', -- low, medium, high, urgent
    
    -- Submission info
    submitted_at TIMESTAMP,
    submitted_by UUID REFERENCES users(id),
    
    -- Processing info
    assigned_to UUID REFERENCES users(id), -- admin/staff
    reviewed_at TIMESTAMP,
    quoted_at TIMESTAMP,
    approved_at TIMESTAMP,
    
    -- Metadata
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    created_by UUID REFERENCES users(id),
    updated_by UUID REFERENCES users(id),
    
    -- Additional info
    internal_notes TEXT,
    customer_feedback_score INTEGER CHECK (customer_feedback_score >= 1 AND customer_feedback_score <= 5)
);

-- Personal information extension
CREATE TABLE request_personal_info (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES package_requests(id) ON DELETE CASCADE,
    
    -- Personal details
    full_name VARCHAR(255) NOT NULL,
    nik VARCHAR(20) UNIQUE,
    birth_date DATE NOT NULL,
    gender VARCHAR(10) CHECK (gender IN ('L', 'P')),
    blood_type VARCHAR(5),
    
    -- Contact information
    phone_primary VARCHAR(20) NOT NULL,
    phone_secondary VARCHAR(20),
    email VARCHAR(255) NOT NULL,
    
    -- Address
    address_street TEXT,
    address_city VARCHAR(100),
    address_province VARCHAR(100),
    address_postal_code VARCHAR(10),
    address_country VARCHAR(100) DEFAULT 'Indonesia',
    
    -- Additional info
    marital_status VARCHAR(20),
    occupation VARCHAR(100),
    emergency_contact_name VARCHAR(255),
    emergency_contact_phone VARCHAR(20),
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Travel preferences
CREATE TABLE request_travel_preferences (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES package_requests(id) ON DELETE CASCADE,
    
    -- Trip details
    number_of_pilgrims INTEGER NOT NULL,
    departure_date DATE NOT NULL,
    return_date DATE,
    duration_days INTEGER,
    departure_city VARCHAR(100) NOT NULL,
    
    -- Package preferences
    package_category VARCHAR(20) CHECK (package_category IN ('hemat', 'standard', 'premium', 'vip', 'custom')),
    hotel_star_rating INTEGER CHECK (hotel_star_rating >= 1 AND hotel_star_rating <= 5),
    
    -- Special requirements
    dietary_requirements TEXT,
    accessibility_needs TEXT,
    medical_conditions TEXT,
    special_requests TEXT,
    
    -- Activities and add-ons
    preferred_activities TEXT[],
    additional_services TEXT[],
    group_preferences TEXT,
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Budget information
CREATE TABLE request_budget_info (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES package_requests(id) ON DELETE CASCADE,
    
    -- Budget details
    max_budget DECIMAL(15,2),
    currency VARCHAR(3) DEFAULT 'IDR',
    
    -- Payment preferences
    payment_method VARCHAR(20) CHECK (payment_method IN ('full', 'installment')),
    payment_schedule TEXT,
    
    -- Budget breakdown (optional)
    accommodation_budget DECIMAL(15,2),
    transportation_budget DECIMAL(15,2),
    meals_budget DECIMAL(15,2),
    activities_budget DECIMAL(15,2),
    other_budget DECIMAL(15,2),
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Document management
CREATE TABLE request_documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES package_requests(id) ON DELETE CASCADE,
    
    -- Document details
    document_type VARCHAR(50) NOT NULL,
    file_name VARCHAR(255) NOT NULL,
    file_path TEXT NOT NULL,
    file_size BIGINT,
    file_mime_type VARCHAR(100),
    
    -- Processing status
    upload_status VARCHAR(20) DEFAULT 'pending', -- pending, processing, verified, rejected
    verification_notes TEXT,
    verified_by UUID REFERENCES users(id),
    verified_at TIMESTAMP,
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Status history tracking
CREATE TABLE request_status_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES package_requests(id) ON DELETE CASCADE,
    
    -- Status change info
    from_status VARCHAR(20),
    to_status VARCHAR(20) NOT NULL,
    changed_by UUID REFERENCES users(id) NOT NULL,
    change_reason TEXT,
    
    -- Additional context
    ip_address INET,
    user_agent TEXT,
    
    created_at TIMESTAMP DEFAULT NOW()
);

-- Quotations/Offers
CREATE TABLE request_quotations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    request_id UUID REFERENCES package_requests(id) ON DELETE CASCADE,
    
    -- Quotation details
    quotation_number VARCHAR(50) UNIQUE NOT NULL,
    total_amount DECIMAL(15,2) NOT NULL,
    currency VARCHAR(3) DEFAULT 'IDR',
    
    -- Validity
    valid_until DATE NOT NULL,
    sent_at TIMESTAMP,
    responded_at TIMESTAMP,
    
    -- Response
    response_status VARCHAR(20) DEFAULT 'pending', -- pending, accepted, rejected, expired
    response_notes TEXT,
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 3. Indexes untuk Performance
```sql
-- Primary lookup indexes
CREATE INDEX idx_package_requests_user_id ON package_requests(user_id);
CREATE INDEX idx_package_requests_status ON package_requests(status);
CREATE INDEX idx_package_requests_submitted_at ON package_requests(submitted_at);
CREATE INDEX idx_package_requests_assigned_to ON package_requests(assigned_to);

-- Compound indexes for common queries
CREATE INDEX idx_package_requests_status_submitted ON package_requests(status, submitted_at);
CREATE INDEX idx_package_requests_assigned_status ON package_requests(assigned_to, status);

-- Full-text search indexes
CREATE INDEX idx_request_personal_info_search ON request_personal_info USING gin(to_tsvector('indonesian', full_name));
CREATE INDEX idx_package_requests_search ON package_requests USING gin(to_tsvector('indonesian', internal_notes));

-- Status history indexes
CREATE INDEX idx_status_history_request_id ON request_status_history(request_id);
CREATE INDEX idx_status_history_created_at ON request_status_history(created_at);

-- Document indexes
CREATE INDEX idx_request_documents_type ON request_documents(document_type);
CREATE INDEX idx_request_documents_status ON request_documents(upload_status);
```

### 4. Constraints dan Triggers
```sql
-- Ensure proper status transitions
CREATE OR REPLACE FUNCTION validate_status_transition()
RETURNS TRIGGER AS $$
BEGIN
    -- Define valid status transitions
    IF OLD.status = 'draft' AND NEW.status NOT IN ('submitted', 'draft') THEN
        RAISE EXCEPTION 'Invalid status transition from % to %', OLD.status, NEW.status;
    ELSIF OLD.status = 'submitted' AND NEW.status NOT IN ('under_review', 'rejected', 'submitted') THEN
        RAISE EXCEPTION 'Invalid status transition from % to %', OLD.status, NEW.status;
    ELSIF OLD.status = 'under_review' AND NEW.status NOT IN ('quoted', 'rejected', 'under_review') THEN
        RAISE EXCEPTION 'Invalid status transition from % to %', OLD.status, NEW.status;
    ELSIF OLD.status = 'quoted' AND NEW.status NOT IN ('approved', 'rejected', 'quoted') THEN
        RAISE EXCEPTION 'Invalid status transition from % to %', OLD.status, NEW.status;
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_validate_status_transition
    BEFORE UPDATE ON package_requests
    FOR EACH ROW
    EXECUTE FUNCTION validate_status_transition();

-- Auto-update updated_at timestamp
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_package_requests_updated_at
    BEFORE UPDATE ON package_requests
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER trigger_update_request_personal_info_updated_at
    BEFORE UPDATE ON request_personal_info
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();

-- Similar triggers for other tables...
```

### 5. Data Validation dan Business Rules
```sql
-- Ensure departure date is in the future
ALTER TABLE request_travel_preferences 
ADD CONSTRAINT check_future_departure 
CHECK (departure_date > CURRENT_DATE);

-- Ensure return date is after departure date
ALTER TABLE request_travel_preferences 
ADD CONSTRAINT check_return_after_departure 
CHECK (return_date IS NULL OR return_date > departure_date);

-- Ensure number of pilgrims is positive
ALTER TABLE request_travel_preferences 
ADD CONSTRAINT check_positive_pilgrims 
CHECK (number_of_pilgrims > 0);

-- Ensure budget is positive
ALTER TABLE request_budget_info 
ADD CONSTRAINT check_positive_budget 
CHECK (max_budget > 0);
```

## Kriteria Keberhasilan

### Technical Requirements
- [ ] Semua tables ter-create dengan benar
- [ ] Relationships antar table berfungsi dengan proper foreign keys
- [ ] Indexes meningkatkan query performance minimal 50%
- [ ] Triggers berfungsi untuk data consistency
- [ ] Constraints mencegah invalid data insertion

### Performance Requirements
- [ ] Simple SELECT query < 100ms untuk 10,000 records
- [ ] Complex JOIN query < 500ms untuk 10,000 records
- [ ] INSERT operation < 50ms
- [ ] UPDATE operation < 100ms

### Data Integrity Requirements
- [ ] No orphaned records (referential integrity)
- [ ] Consistent data types dan formats
- [ ] Proper handling of NULL values
- [ ] Audit trail lengkap untuk semua perubahan

### Security Requirements
- [ ] Sensitive data ter-enkripsi (NIK, email)
- [ ] Proper access control di database level
- [ ] Audit logging untuk sensitive operations
- [ ] Backup dan recovery procedures
